# 01. 要件定義

## 1.1 背景・目的

Web を見ていると「あとで読みたい」記事に頻繁に出会うが、ブラウザのブックマークや
タブに溜め込むだけだと、二度と読まれずに埋もれてしまう。
本アプリは「保存 → 読む → 整理」の一連を 1 つのスマホアプリで完結させ、
保存した記事を「ちゃんと読める・あとで見つけられる」状態に保つことを目的とする。

参考イメージは **Readwise Reader**（保存・リーダービュー・ハイライト・整理を備えた read-it-later アプリ）。
本アプリはその中核体験のうち「保存」と「整理」を MVP として最小構成で実現する。

## 1.2 ターゲットユーザー

- 主に **本人（個人利用）** を想定。まずは自分が毎日使えることを最優先。
- 想定行動: スマホで記事・ニュース・ブログを見る → その場では時間がない →
  あとでまとめて読みたい → 読んだ／読みたいものをテーマ別に整理したい。

## 1.3 解決したい課題

| 課題 | 本アプリでの解決 |
| --- | --- |
| 保存先がバラバラ（ブックマーク／タブ／LINE 送信など） | 共有シートから 1 タップで集約保存 |
| 保存したことを忘れる・見つけられない | タグ・フォルダ・検索で整理 |
| 記事ページが広告やナビで読みにくい | 本文抽出によるリーダービュー（※簡易版を MVP に含む） |
| 既読／未読が分からない | 既読フラグ・アーカイブで状態管理 |

## 1.4 スコープ

### MVP（このフェーズで作る）

- **記事の保存**
  - 他アプリの共有シート（`ACTION_SEND` / text/plain）から URL を受け取って保存
  - アプリ内で URL を直接貼り付けて保存
  - 保存時にタイトル・サムネイル・本文を自動取得（メタデータ + 本文抽出）
- **記事の閲覧**
  - 一覧表示（保存日時順、サムネイル＋タイトル＋ドメイン＋抜粋）
  - 記事詳細 = リーダービュー（抽出本文を読みやすく表示）。取得失敗時は元 URL を WebView/ブラウザで開く導線
  - 既読／未読、アーカイブの状態切り替え
- **整理**
  - タグの付与・削除（複数可）、タグでの絞り込み
  - フォルダ（1 記事 = 1 フォルダ）への振り分け
  - キーワード検索（タイトル・本文・タグ対象）
- **保存先**
  - 端末ローカルのみ（オフラインで閲覧可）。クラウド同期は MVP 対象外

### MVP に含めない（将来候補）

- ハイライト・メモ（注釈機能） … Readwise の目玉だが次フェーズ
- クラウド同期 / 複数端末 / バックアップ
- アカウント・ログイン
- RSS / ニュースレター取り込み
- 全文検索の高度化（あいまい検索・並べ替え高度化）
- 読み上げ（TTS）、PDF / EPUB 取り込み
- iOS 版・Web 版

## 1.5 制約・前提

- ネットワーク必須なのは「保存時の取得」のみ。閲覧はオフライン可。
- 本文抽出は対象サイトにより精度差が出る。失敗時は元 URL フォールバックで体験を担保する。
- 個人開発・ローカル完結のため、サーバー／認証は持たない（運用コストゼロを志向）。

## 1.6 非機能要件

| 項目 | 目標 |
| --- | --- |
| パフォーマンス | 一覧スクロールは 60fps、保存処理は UI をブロックしない（バックグラウンド取得） |
| オフライン | 保存済み記事は通信なしで閲覧可能 |
| データ永続化 | アプリ再起動・OS 再起動後もデータ保持（Room） |
| プライバシー | データは端末内のみ。外部送信は記事取得時の対象サイトアクセスのみ |
| 対応 OS | Android 8.0 (API 26) 以上 |
| 言語 | UI は日本語（将来 i18n 可能な構造） |

## 1.7 成功の目安（MVP）

- 共有シート経由で 3 タップ以内に記事保存が完了する
- 保存した記事が一覧→詳細で問題なく読める（主要サイトで本文抽出が概ね機能）
- タグ／フォルダ／検索で目的の記事に 10 秒以内でたどり着ける
- 本人が 2 週間継続して使える
