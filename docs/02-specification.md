# 02. 機能仕様

各機能の振る舞いを「ユーザー操作 → システムの動作 → 結果」で定義する。
画面の詳細は [05-ui-screens.md](05-ui-screens.md)、データ構造は [04-data-model.md](04-data-model.md) を参照。

## F-1. 記事の保存（URL 取り込み）

### F-1.1 共有シートからの保存

- **トリガー**: 他アプリ（ブラウザ、SNS、ニュースアプリ等）の「共有」から本アプリを選択。
- **受け取り**: `Intent.ACTION_SEND` + `text/plain`。共有テキスト中の最初の URL を抽出する。
- **動作**:
  1. URL を即座に DB へ「保存中（PENDING）」状態でレコード作成（楽観的保存）。
  2. すぐに保存完了トースト／確認 UI を返し、共有元アプリに戻れるようにする。
  3. バックグラウンド（WorkManager）でメタデータ＋本文取得を実行し、レコードを更新。
- **重複**: 同一 URL（正規化後）が既に存在する場合は新規作成せず、既存記事を更新日時で前に出す。

### F-1.2 アプリ内での URL 貼り付け保存

- 一覧画面の「＋」ボタン → URL 入力ダイアログ → 保存。動作は F-1.1 と同じ取得フローに合流。
- クリップボードに URL がある場合は入力欄に候補として提示（任意）。

### F-1.3 メタデータ・本文取得

- 取得対象:
  - **タイトル**: `<title>` / OGP `og:title`
  - **サムネイル**: OGP `og:image`
  - **サイト名 / ドメイン**: `og:site_name` / ホスト名
  - **本文**: 本文抽出ライブラリ（Readability 系）で主要コンテンツを抽出し HTML/プレーンに整形
  - **抜粋**: 本文先頭 or `og:description`
  - **推定読了時間**: 本文文字数 ÷ 想定読速（例: 日本語 500 字/分）で算出
- **失敗時**: 本文抽出に失敗しても、最低限 URL とタイトル（取れれば）で保存。
  `extractionStatus = FAILED` を記録し、詳細画面では元 URL を開くフォールバックを提示。
- **状態遷移**: `PENDING`（作成直後）→ `SUCCESS` or `FAILED`（取得完了後）。

### F-1.4 URL 正規化（重複判定用）

- スキーム小文字化、`http`→`https` 寄せ（任意）、末尾スラッシュ正規化、
  トラッキングパラメータ（`utm_*`, `fbclid`, `gclid` 等）の除去。
- 正規化後 URL をユニークキー扱いにする。

## F-2. 記事の閲覧

### F-2.1 一覧表示

- 既定表示: **受信トレイ（未アーカイブ）** を保存日時の降順。
- 各行: サムネイル、タイトル、ドメイン、推定読了時間、保存日時、既読/未読インジケータ、タグチップ。
- 取得中（PENDING）の記事は「取得中…」表示。FAILED は控えめなエラー表示＋再取得導線。
- 表示フィルタ: 受信トレイ / 既読 / アーカイブ / フォルダ別 / タグ別（[F-3]）。
- 並べ替え: 保存日時（既定）/ 更新日時 /（将来）読了時間。

### F-2.2 記事詳細（リーダービュー）

- 抽出本文を読みやすいタイポグラフィで表示（フォントサイズ・行間・テーマは設定で調整可、後述 F-5）。
- ヘッダー: タイトル、サイト名、元 URL を開くボタン、共有ボタン。
- 取得失敗 or 抽出が不十分な場合: 「元のページを開く」ボタンで WebView / 外部ブラウザ表示。
- 開いた時点で既読フラグを自動 ON（設定で無効化可）。

### F-2.3 状態管理

- **既読/未読**: 手動トグル＋詳細を開いた際の自動既読。
- **アーカイブ**: 読み終えた／不要になった記事を受信トレイから外す（削除はしない）。
- **削除**: 完全削除（確認ダイアログあり）。一覧からスワイプで「アーカイブ」、長押しメニューで「削除」。
- **お気に入り（任意）**: スター付与で後から参照しやすく（MVP では実装簡易、後回し可）。

## F-3. 整理（タグ / フォルダ / 検索）

### F-3.1 タグ

- 1 記事に **複数タグ**（多対多）。詳細画面・一覧の長押しメニューから付与/解除。
- タグは自由入力で新規作成、既存タグはサジェスト。タグ一覧から記事を絞り込み可能。
- タグ名は正規化（前後空白除去、大文字小文字・全半角の扱いを統一）して重複作成を防ぐ。

### F-3.2 フォルダ

- 1 記事は **1 フォルダ**に所属（未所属＝受信トレイ）。フォルダは作成・リネーム・削除可。
- フォルダ削除時、中の記事は「未所属（受信トレイ）」に戻す（記事自体は消さない）。
- ナビゲーションドロワー or タブからフォルダ一覧 → フォルダ内記事一覧へ。

> タグとフォルダの使い分け: フォルダは「置き場所（排他的）」、タグは「横断的なラベル（複数可）」。

### F-3.3 検索

- 対象: タイトル / 抜粋 / 本文 / タグ名 / ドメイン。
- 方式: Room の `LIKE` 検索（MVP）。将来 FTS4/FTS5 で全文検索へ拡張。
- 検索結果は一覧 UI を再利用。フィルタ（既読/アーカイブ等）と併用可能。

## F-4. 保存・同期

- ストレージ: 端末ローカルの Room データベースのみ。
- 同期・バックアップは MVP 対象外（将来: エクスポート/インポート JSON → クラウド同期）。

## F-5. 設定（MVP は最小限）

- リーダービュー: フォントサイズ、行間、テーマ（ライト/ダーク/システム追従）。
- 動作: 「開いたら自動で既読にする」ON/OFF。
- データ: 全データ削除、（将来）エクスポート。

---

## F-6. エラー / エッジケース

| ケース | 振る舞い |
| --- | --- |
| 共有テキストに URL が無い | エラートースト「URL が見つかりません」、保存しない |
| ネットワーク無しで保存 | PENDING で保存し、オンライン復帰時に WorkManager が再取得 |
| 本文抽出失敗 | FAILED 記録、詳細で元 URL フォールバック＋再取得ボタン |
| 重複 URL | 既存記事を更新（前面に）し、重複作成しない |
| 取得が極端に遅い/巨大ページ | タイムアウト（例 15 秒）、サイズ上限を設けて打ち切り |
| 不正な URL | バリデーションで弾く |
