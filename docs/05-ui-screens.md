# 05. 画面設計 / 画面遷移

Material 3 + Jetpack Compose 前提のワイヤーフレーム（テキスト版）。

## 5.1 画面一覧

| ID | 画面 | 役割 |
| --- | --- | --- |
| S-1 | 記事一覧（受信トレイ） | ホーム。保存記事のリスト |
| S-2 | 記事詳細（リーダービュー） | 抽出本文を読む |
| S-3 | URL 追加ダイアログ | アプリ内で URL を貼り付け保存 |
| S-4 | フォルダ一覧 / フォルダ内 | フォルダで整理・閲覧 |
| S-5 | タグ一覧 / タグ絞り込み | タグで整理・閲覧 |
| S-6 | 検索 | キーワード検索 |
| S-7 | 設定 | 表示・動作設定 |
| S-0 | 共有受信（UI なし/軽量確認） | 共有シートからの保存受け口 |

## 5.2 画面遷移図

```
        [共有シート(他アプリ)]
               │ ACTION_SEND
               ▼
        S-0 共有受信 ──保存──▶ (トースト) ─▶ 共有元へ戻る
                                   │
                                   ▼ (アプリ起動時)
  ┌──────────────── S-1 記事一覧(受信トレイ) ◀── ホーム ──┐
  │   ├─ FAB[＋] ─▶ S-3 URL追加ダイアログ                 │
  │   ├─ 記事タップ ─▶ S-2 記事詳細(リーダー)              │
  │   ├─ 上部検索 ─▶ S-6 検索                              │
  │   └─ ドロワー/タブ ─▶ S-4 フォルダ / S-5 タグ / S-7 設定 │
  └────────────────────────────────────────────────────────┘
```

ナビゲーション: ボトムナビ or ナビゲーションドロワーで
「受信トレイ / フォルダ / タグ / 検索 / 設定」を切り替える（MVP はドロワー想定）。

## 5.3 各画面の要素

### S-1 記事一覧（受信トレイ）
- **トップバー**: アプリ名、検索アイコン、フィルタ（受信/既読/アーカイブ）。
- **リスト行（カード）**:
  ```
  ┌───────────────────────────────────────┐
  │ [サムネ] タイトル（最大2行）            │
  │         example.com ・ 5分 ・ 6/7      │
  │         #タグ #タグ        ● 未読      │
  └───────────────────────────────────────┘
  ```
  - PENDING: 「取得中…」プレースホルダ。FAILED: 「取得失敗 ↻再取得」。
- **操作**: タップ=詳細、左スワイプ=アーカイブ、長押し=メニュー（タグ/フォルダ/既読/削除）。
- **FAB**: 「＋」URL 追加（S-3）。
- **空状態**: 「保存した記事がここに表示されます。共有から追加してみましょう」。

### S-2 記事詳細（リーダービュー）
- **トップバー**: 戻る、元ページを開く（↗）、共有、メニュー（タグ/フォルダ/アーカイブ/削除）。
- **本文**: タイトル → サイト名・読了時間 → 抽出本文（読みやすいタイポ）。
- 抽出失敗時: 「この記事は本文を取得できませんでした」＋「元のページを開く」ボタン。
- 設定のフォントサイズ／行間／テーマを反映。開いた時点で既読に。

### S-3 URL 追加ダイアログ
- URL 入力欄（クリップボード候補を自動提示）、保存ボタン、キャンセル。
- 保存で取得フロー（F-1.3）に合流。

### S-4 フォルダ
- フォルダ一覧（名前＋件数）。新規作成 FAB、リネーム/削除（長押し）。
- フォルダ選択 → 中の記事一覧（S-1 と同じ行 UI を再利用）。

### S-5 タグ
- タグ一覧（名前＋件数、チップ表示）。タグ選択 → 絞り込み一覧。

### S-6 検索
- 検索バー＋結果リスト（S-1 行 UI 再利用）。フィルタ併用可。入力に応じてインクリメンタル表示。

### S-7 設定
- 表示: フォントサイズ、行間、テーマ（ライト/ダーク/システム）。
- 動作: 開いたら自動既読 ON/OFF。
- データ: 全データ削除（確認あり）。アプリ情報/バージョン。

## 5.4 デザイン指針

- Material 3、ダイナミックカラー対応（Android 12+）。ダークモード対応。
- 「読む」体験優先で、リーダービューは余白・行間を広めに。
- 主要操作（保存・既読・整理）は最短タップで届く位置に置く。
- 日本語 UI、フォントは可読性重視（システムフォント）。

## 5.5 状態表示の一貫ルール

| 状態 | 表現 |
| --- | --- |
| 取得中 | スケルトン or 「取得中…」ラベル |
| 取得失敗 | エラーチップ＋再取得アクション |
| 空 | イラスト/文言で次アクションを促す |
| 既読 | タイトルを淡色化＋既読バッジ非表示 |
| オフライン | 保存済みは通常表示。新規取得時のみバナー通知 |
